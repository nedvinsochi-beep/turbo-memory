# MEMORY_PROTOCOL — протокол обновления памяти

Дата создания: 2026-06-29

## Назначение

Этот файл описывает, как обновлять общую память Михаила Гелаша и его AI-систем.

Главный принцип: важные факты не должны жить только в памяти одного AI-чата. Всё значимое фиксируется в GitHub.

## Что обязательно фиксировать

1. Новые проекты.
2. Изменения в стратегии Cosmos Realty.
3. Изменения в ГиЯр / GEAR.
4. Новые роли, сотрудники, партнёры, подрядчики.
5. Решения по CRM, Bitrix24, AI-агентам, Telegram-ботам.
6. Скрипты продаж, офферы, воронки, KPI.
7. Юридические решения.
8. Промпты и инструкции для Claude, ChatGPT, Codex.
9. Ошибки, которые нельзя повторять.

## Уровни достоверности

### Факт
Информация явно подтверждена Михаилом, документом, скриншотом, CRM, репозиторием или таблицей.

### Рабочая гипотеза
Информация логично следует из обсуждения, но не подтверждена напрямую.

### Черновик
Материал создан AI и требует проверки Михаилом.

## Как обновлять память

1. Если появилась новая важная информация — добавить её в соответствующий Markdown-файл.
2. Если изменился проект — обновить файл проекта.
3. Если появилось важное решение — добавить строку в `MEMORY_LOG.md`.
4. Если появилась новая инструкция по стилю — обновить `STYLE/voice-and-tone.md`.
5. Если появился новый агент — добавить файл в `AI_AGENTS/`.

## Формат записи в MEMORY_LOG.md

```md
## 2026-06-29

- Решение: GitHub используется как внешний источник правды для AI-памяти.
- Причина: память внутри Claude/ChatGPT не переносится и может быть потеряна.
- Последствие: ключевые знания фиксируются в Markdown-файлах.
```

## Правило для AI

Перед тем как предлагать создать что-то новое, AI обязан проверить:
- есть ли уже такой репозиторий;
- есть ли уже такой файл;
- есть ли уже похожая структура;
- не было ли это ранее решено.

Если доступа нет — AI должен честно сказать, что не видит текущее состояние.

## Правило источника правды

При конфликте источников приоритет такой:

1. Прямое указание Михаила в текущем разговоре.
2. Актуальный файл в GitHub.
3. Документ/таблица/CRM/скриншот.
4. Память AI.
5. Предположение AI.

## Что не делать

- Не плодить новые репозитории без необходимости.
- Не дублировать память в десяти местах.
- Не считать память Claude или ChatGPT единственным источником правды.
- Не хранить токены, пароли, ключи API и PAT в Markdown-файлах.
- Не публиковать секреты в чатах.
